# CellState Workbench — interactive demo (UI + harness)

The interactive front-end for the unified cell-state world model. You describe a
cell state in plain language; Claude turns it into a plan; the trained model
**generates the cell's morphology from the learned state S** on a GPU. You can then
walk the state, intervene on it, and interrogate the result.

Every cell shown here is **generated by the real CFG diffusion decoder from S** —
there is no procedural or fake renderer in this app.

## Architecture

```
browser (React, :5210)
        │  only ever talks to the local proxy
        ▼
local harness proxy (Node, :8787)          ← holds the Claude API key
        ├── POST /api/interpret  → Claude   (free-text state → plan JSON)
        ├── POST /api/ask        → Claude   (annotation replies, grounded in the numbers)
        └── GET  /api/cell | trajectory | intervene | generate
                                 → Modal    (encoder E + CFG diffusion decoder, GPU)
```

The proxy exists for two reasons, both load-bearing:

1. **The API key must never reach the browser.** A React bundle is public; a key
   shipped in it is a leaked key. The proxy keeps it server-side, read from `.env`.
2. **It removes CORS.** The browser cannot call the Modal endpoints cross-origin;
   the proxy calls them server-to-server and relays the JSON.

## What the UI does

| Panel | Function |
|---|---|
| **Sessions** (left) | independent working sessions, each with its own objects |
| **Viewport** (centre) | real cell (left) · **generated from state S** (centre) · real target (right) |
| **Scrubber** | walk the state; `ΔS` is the distance travelled in state space |
| **Console** (bottom) | `state>` free text → Claude plan → model generation, with a log |
| **Inspector · State** | the plan Claude produced, plus live state numbers |
| **Inspector · Annot** | coordinate-anchored annotation threads on the generated image |
| **Inspector · Intervene** | push the state mid-trajectory toward another cell, and regenerate |

**Annotations** — click anywhere on the generated cell, write a note, and a numbered
pin appears. Click the pin to open its thread and ask Claude about that spot. Claude
answers **grounded in that cell's actual numbers** (the plan, ΔS, and the validated
correlations) and is instructed never to invent numbers.

**Intervention** — from frame `at` onward, the state is blended toward another cell's
state with weight `strength`, then re-decoded:

```
t <  at :  S(t) = (1 - α)·S_start + α·S_end
t >= at :  S(t) = (1 - strength)·S(t) + strength·S_toward
```

Verified empirically: `strength = 0` reproduces the plain trajectory byte-for-byte,
and every frame before `at` is untouched.

## Run

```bash
npm install
cp .env.example .env      # add your ANTHROPIC_API_KEY (.env is gitignored)
npm run proxy             # terminal 1 — harness on :8787
npm run dev               # terminal 2 — app on :5210
```

Optional `.env` keys: `MODAL_BASE`, `ANTHROPIC_MODEL`, `PROXY_PORT`.

Try `an epithelial cell that undergoes EMT over time` — it produces a multi-frame
trajectory, which enables both the scrubber and the intervention panel.

## Honest notes (read before demoing)

- **The scrubber is not a learned clock.** `/trajectory` is a *linear interpolation
  in S between two real cells* — we confirmed the per-step distances are exactly
  proportional to the interpolation weight. It shows that the state space is smooth
  and decodable everywhere; it does **not** simulate mitosis or differentiation. The
  project's validated time axis is a separate experiment (an independent EMT
  time-course projected through the frozen encoder, Spearman rho = 0.50).
  Cells that appear to **merge** during a scrub are a **crossfade artifact** between
  two different cells — and the 64-pixel patches contain neighbouring nuclei. It is
  not a biological prediction.
- **Text → cell is not phenotype-aware yet.** The harness maps the wording to a
  *deterministic* cell index; it does not retrieve a cell that actually expresses the
  described markers. Making that real needs a marker-gene lookup endpoint over the
  expression matrix — a small addition on the model side, not a redesign.
- **Cell ids index the 200,000-cell dataset** (0–199,999, taken modulo 200,000). An
  id carries no phenotype meaning on its own — see the point above.
- **The first request cold-starts the GPU** (~30–60 s); after that, generation is
  ~25 s. Generation is deterministic: identical parameters return identical frames.

What *is* validated: images are generated from S and are **cell-specific** — they
match their own cell better than a random one (gap +0.17) — with size and brightness
tracking the real cell at r ≈ 0.66–0.68, against a shuffle control of −0.09. Full
details in the paper under [`../report/`](../report/).
